    Research & Projects

    • Computer Program Complexity and Its Correlation with Program Features and Sociolinguistics.
      M.Sc. in Computer Science Thesis Work (2018-2020)
      Supervisor: Professor Dr. Jackie Rice

      Overview: The use of natural language among humans depends on factors such as social status, economic status, gender, age, and region. These factors may also impact the use of programming languages. This research focuses on the influence of socio-characteristics, especially region and gender, on the use of programming languages. A set of programming features was used in machine learning experiments to determine whether there are any similarities in the use of C++ programming language based on the programmer's region or gender. Frequency analysis was previously carried out to identify any gender-based differences in how features were used by the authors. We performed a similar type of analysis to determine which features are more or less likely to be used in the programs from a particular region or written by a particular gender. We also explored programming complexity based on a previous work where researchers described a software readability model that included programming features such as keywords and operators. Those features were found to be strongly correlated with three software quality measures: software defect log messages, automated defect reports, and code change. In our work, two types of complexity metrics were used to determine a correlation between programming features and complexity features. Our goal is to determine whether these features may make computer programs easier or harder to read, or more or less complex (and hence more or less prone to bugs).

    • RoleRecommender: A Recommendation System that recommends commenter role (developer, tester/QA) based on Bug Report Comments.
      M.Sc. in Computer Science Project Work (2018)
      Supervisor: Assistant Professor Dr. John Anvik

      Overview: Analysis of bug reports and bug report comments largely depends on many machine learning techniques. This paper focuses on those machine learning techniques for recommending commenter role by analyzing bug reports comments. My main goal was to create a recommendation system that can recommend whether a commenter is a developer or tester. Finding a commenter role is very important as it will help design role-specific IDEs, detect plagiarism, and improve communication between developers and testers. In this work, I used Mozilla bug report comments as the data set. These comments were written using plain text. I extracted the features from those comments. Then, I used a classifier for training and testing purposes. I divided the data set into two parts. I trained the classifier with one part. Then, I evaluated the performance of the recommender system by testing with another part. I also performed a random role recommendation approach where I randomly selected the commenter role. After that, I evaluated the performance of the random role recommendation approach. Finally, I compared the performance of the random role recommendation approach and the recommendation system. The results of various experiments are very promising. The results showed that the recommendation system achieved precision of 87.5%, recall of 77.78%, and F-measure of 82.35%, whereas the random role recommendation approach was able to achieve precision of 57.14%, recall of 44.44%, and F-measure of 50.0%. Therefore, the recommendation system outperforms the random role recommendation approach by improving the precision by 30.36%, recall by 32.74%, and F-measure by 32.35%.

    • A Semantic Approach to Object Detection Algorithm Selection
      B.Sc. in Computer Science and Engineering Thesis Work (2016-2017)
      Supervisor: Associate Professor Dr. Khandaker Tabin Hasan

      Overview: This research focused on the selection of image detection algorithms following the semantic approach. Finding algorithms is very hard when anyone tries to detect an object from an image. Though there are many object detection algorithms, there is no such way to find out which algorithm will give better results without running them all one by one which is very time-consuming. We proposed an idea by which we can select the most suitable image detection algorithm following the semantic approach. A lexical database “WordNet” was used to differentiate the type of object we are dealing with. Then we developed a hierarchical tree where, an object, particularly a “noun” can be fit into. We computed the most occurring “sense” among all the “senses” to place a noun in the tree node. After placing an object in its appropriate node, we can find out the algorithm that suits best for detecting similar types of objects. Four different image detection algorithms, WordNet python API, and OpenCv were used for grouping words and running algorithms on images.
